## 2026-06-04 — aiMatchFaqId Fix + Frontend #8

### Bug: `aiMatchFaqId` never saved on question create

**Root cause (2-part bug):**
1. Shape 2 (AI match) in `questions.controller.ts` returned `{ aiMatch, faq }` to frontend but never called `create()` — question was never persisted, matched FAQ ID was discarded
2. Shape 1 and forceSubmit paths referenced `matchedFaqId` which was a completely undefined variable → `ReferenceError`

**Fix — `questions.controller.ts`:**
- Shape 2 now captures `intentOrMatch.faq.id` and calls `questionsService.create(dto, userId, capturedFaqId)` — persists question with `aiMatchFaqId` set
- Shape 1 and forceSubmit paths no longer pass `matchedFaqId` (undefined)

**Fix — `ask.tsx`:**
- `submissionMutation.onSuccess` now receives `payload` as second arg (TanStack Query injects it)
- On AI match, `setPendingPayload(payload)` stores the original `{ title, body, category, tags }` so reject → force-submit posts correct data

**Bonus fix:** `questions.service.ts` line 59 had a stray `\n` literal (corruption artifact) — fixed to proper newline, `tsc --noEmit` now exit 0

### Files Changed
- `backend/src/questions/questions.controller.ts` — Shape 2 captures `capturedFaqId`, passes to `create()`; Shape 1/forceSubmit simplified
- `backend/src/questions/questions.service.ts` — `\n` literal removed
- `frontend/src/routes/ask.tsx` — `onSuccess` now captures `payload` arg and sets `pendingPayload` on AI match

### Status
- `tsc --noEmit` ✅ exit 0 both frontend and backend
- `aiMatchFaqId` saved on every question create (including AI-match and force-submit paths)
- Frontend reject flow posts original form payload, not matched FAQ object
